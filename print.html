Interp, the Language

Interp is currently in beta testing! Don't use it for production!

Interp is a procedural general-purpose dynamically typed interpreted scrpiting language. Although it uses duck-typing, it also enforces type safety. The interplang interpreter is coded entirely in Rust, and the language can be extended by dynamic foreign functions programmed in pure Rust. Functions in Interp are first-class-citizens and can be assigned to variables and passed as arguments to other functions. Interp supports both 64-bit integers as well as 64-bit floating point numbers. The keytable in Interp is a map of sorts, with keys and values, these can be of whatever type. There are no classes, It's not an object oriented language!

Hello, world!

Execute a source-code file

You can run Interp files (*.it) with the interp binary. Create a file called hello.it:

Print("Hello, world!");

Then just simply run the command:

$ interp hello.it
Hello, world!

Repl

the interp binary can also spin up an interactive session (a Read-Eval-Print-Loop).

$ interp
> Print("Hello, world!")
Hello, world!

To execute a file simply use imp.

> imp "hello.it"
Hello, world!

Fibbonacci Sequence

fun Fib(n Int[0..]) Int[0..] {
    if n <= 1 {
        ret 1;
    }
    ret Fib(n - 1) + Fib(n - 2);
}

Print(Fib(9));
# output: 55

Birthday

struct Person {
    first_name String,
    last_name String,
    age Int[0..],
    birthday Function<Person, Void>
}

fun Person(first_name String, last_name String, age Int[0..]) Person {
    var obj = { first_name=first_name, last_name=last_name, age=age };
    fun Birthday(self Person) {
        self.age += 1;
    }
    obj.birthday = Birthday;
    ret obj;
}

var samuel = Person(
    "Samuel",
    "Kriikkula",
    19
);

Print(samuel.age); # 19
samuel:birthday();
Print(samuel.age); # 20

Animal

mod animals {

    atom Dog;
    atom Cat;

    enum AnimalType {
        Dog, Cat
    }

    struct Animal {
        type AnimalType,
        name String,
        age  Int[0..],
    }

    fun Animal(type AnimalType, name String, age Int[0..]) Animal {
        var obj = { type=type, name=name, age=age };
        ret obj;
    }

}

var bob = animals::Animal(animals::Cat, "Bob", 2);

Print(bob);
# output: {"type" = Atom("animals::Cat"), "name" = "Bob", "age" = 2}

ForEach

fun ForEach(function Function<Any, Void>, list List) {
    var i = -1;
    while list[i + 1] != void {
        i += 1;
        function(list[i]);
    }
}

for item in ForEach([1, 2, 3]) {
    Print(item);
}

# output:
# 1
# 2
# 3

Values

Interp implements the following primitives. They can be stored into variables as well as they can be passed into functions as arguments.

  • Void
  • Null
  • String
  • Int
  • Float
  • Bool
  • List
  • Keytable
  • Function

Void

Void (keyword void) has two uses.

Denoting 'nonexistent'

When you reference a variable or field that does not exist, you will get Void. This will also usually throw an error, except when checking if something is void.

Assuming variable does not exist.

The following is valid:

if variable == void {
    Print("variable does not exists")
}

The following throws an error:

Print(variable);

<stdin>:1:7:mistaken reference to void. (possible typo)
  |
1 | Print(variable);
  |       ^

As a function return value

Void as a return type means that the function never returns any value at all. If a return type is not specified, it defaults to Void

These are equivalent:

fun PrintHello() {
    Print("Hello");
}

fun PrintHello() Void {
    Print("Hello");
}

Lists

A list can't have Void as an item. When a List index returns Void it means the end of the List.

Any type

Void is the only Value that does not fit in to the Any type.

Null

Null (keyword null) means that the variable exists, but has no value.

As a Type

A common use for Null is in an Or type. For example if you optionally need a String, you can use the or and Null type.

fun TakesOptionalString(str String or Null) {
    Print(str);
}

TakesOptionalString("Hello");
TakesOptionalString(null);

This will print either the string or <null>

In a List

Unlike Void, Lists can contain Null.

String

A String is a sequence of characters. They are encoded in UTF-8.

A String literal is defined like so:

var string = "My String";

Strings can be printed with the Print function.

Print("Hello, world!");

Indexing

Strings can be indexed with Ints, index 0 is the first element.

string[3] # fourth unicode character

Concatenation

You can concatenate Strings with the + operator.

Print("Hello, " + "world!");

Format

Strings can be formatted with the Format function.

Print(Format("{} + {} = {}", 1, 2, 1 + 2));
# Output = 1 + 2 = 3

Int

Ints are 64-bit signed integers.

var int = 123;

Math

The following mathematical operators are supported

1 + 2 # Addition
1 - 2 # Subtraction
1 * 2 # Multiplication
1 / 2 # Division
1 % 2 # Remainder
1 ^ 2 # Power

As a type

Value ranges can be expressed as a type. Int[0..10] accepts an Int that is >= 0 & < 10.

Float

Floats are 64-bit floating-point nubmers.

var float = 1.23;

Math

The following mathematical operators are supported

1 + 2 # Addition
1 - 2 # Subtraction
1 * 2 # Multiplication
1 / 2 # Division
1 % 2 # Remainder
1 ^ 2 # Power

As a type

Value ranges can be expressed as a type. Int[1.2 .. 3.4] accepts a Float that is >= 1.2 & < 3.4.

Bool

Booleans are either true or false.

The most common use is the If statement.

if bool {
    Print("bool == true");
}

Operations

Bools can be crafted with some operators.

a == b        # Equation
a != b        # Negative Equation
a | b         # Or
a & b         # And
a < b         # Less Than
a > b         # More than
a <= b        # Less Than or Equals
a >= b        # More Than or Equals
a is Type     # Fits into Type
a is not Type # Does not Fit into Type

List

Lists are sequences of values.

var list = [1, "str", true, [2.3]];

Indexing

Lists can be indexed with Ints, index 0 is the first element.

list[0]

Negative indicies start from the end, index -1 is the last element.

list[-1]

Concatenation

Lists can be concatenated with the + operator.

[1, 2, 3] + [4, 5, 6] == [1, 2, 3, 4, 5, 6]

Length

List length can be checked with the Len function.

Print(Len([1, 2, 3]));
# output: 3

As a Type

To match all lists, simply use List . To match a list which all values match a type, you can use a generic type argument.

fun TakesIntList(arg List<Int>) {
    Print(arg)
}

Keyable

Keytables (or Dicts) are the most powerful Value in the language. They contain key-value-pairs. Keys and Values can be of any Type.

Keytables can be constructed in a few ways:

var table = {foo = 1, bar = 2};

var other_table = {}
other_table.foo = 1;
other_table.bar = 2;

var third_table = {}
third_table["foo"] = 1;
third_table["bar"] = 2;

# Other than strings can also be keys 
third_table[3] = 3;

Print(third_table);
# output: {"foo" = 1, "bar" = 2, 3 = 3}

KeyValuePairs

You can iterate all field of an object with the KeyValuePairs function.

var table = {a = 1, b = 2};
Print(KeyValuePairs(table));
# output: [{"key" = "a", "value" = 1}, {"key" = "b", "value" = 2}]

Variables

A variable has a name and holds a value. When a variable is referenced but does not exist Void is returned.

Variables are defined with the var keyword.

var variable = "I'm the Value!";

After defining variables can be reassigned. Variables themselves do not have types designated for them.

variable = true;

Variables can be passed into functions. The value is copied.

Function(variable);

Functions can also take the reference of a variable, this way, the value is not copied.

Function(ref variable);

In-place operations

You can short-hand operators when assigning.

variable += 1;
# Is the same as
variable = variable + 1;

Functions

In Interp, functions are first-class-citizens. Functions are defined with the fun keyword. Parameters can be defined with the parameter name followed by a Type. The return type is specified after the parameter list. If no return type is specified, Void is assumed. Values are returned with the ret keyword.

fun MyFunction(param1 Int, param2 String) Bool {
    ret true;
}

This is a function returning Void.

fun PrintHello() {
    Print("Hello, world!");
}

Functions can be assigned to variables.

var my_var = PrintHello;
my_var();
# Output: Hello, world!

Functions can take variables by reference.

var take_ref = 1;
fun AddOne(arg Int) {
    var += 1;
}
Print(take_ref);
# Output: 2

Default and named parameters

Default values for paramters are allowed. Parameters can also be referred with their name.

fun Function(int foo, string bar = "default_string", baz Bool = false) {
    ...
}

Function(1, baz=true) # allowed

Lambda Functions

Lambdas are a shorthand for defining functions.

fun ExecLambda(lambda Any) {
    lambda(1, 2);
}

ExecLambda(|a, b| => Print(a + b));
# Output: 3

As a type

To match all functions, simply use Function.

If the function has a specific return type, use Function<ReturnType>. The arguments can be specified before the return type. If you want a function that takes an Int and returns an Int, you use Function<Int, Int>.

fun TakesFunction(func Function<Int, Int, Int>) {
    Print(func(1, 2));
}

fun Sum(x Int, y Int) Int {
    ret x + y;
}

TakesFunction(Sum);

IO

IO is implemented with some built-in functions.

Standard Streams

  • Print can be used to send text into standard out or standard error.
  • Input is used to get input from the user fXrom standard in.

File IO

  • WriteFile can be used to write text in to a file.
  • ReadFile can be used to read text from a file.
  • WriteBytes can be used to write bytes in to a file.
  • ReadBytes can be used to read bytes from a file.

Process IO

  • Exec can be used to execute a command.

Print Function

Print is a versatile function. Print a value to standard out followed by a newline.

Print(1);

Print also can take a second argument, an object that contains options.

Print(
    1,
    newline=false, # Optional, defaults to true
    stream="stderr" # Optional, "stdout" or "stderr", defaults to "stdout",
);

User Input

User input is gotten with the Input function.

Print("Enter name: ", { newline=false });
var name = Input();
Print(Format("Hello, {}!", name));

Byte IO

Interp defines two functions for reading and writing bytes from and to a file.

WriteBytes("text.txt", [55, 54]);
var bytes = ReadBytes("test.txt");

Exec

With the Exec function you can execute an executable with arguments, and get the standard output as a String.

Print(Exec("echo", "Hello, world!");)
# output: Hello, world!

If Statement

If statements execute the associated code if the condition is true. Else If statement executes if the If condition above is false and the associated condition is true. Else executes if the above If statements are all false.

if a {
    Print("a is true");
} else if b {
    Print("a is false and b is true");
} else {
    Print("a and b are both false");
}

The While Loop

While loops execute the given code in a loop until the condition evaluates to false or a break statement is encountered.

var i = 0;
while i < 10 {
    Print(i);
    i += 1;
}

While loops can be named. brk exits the loop, if not named, the inner-most loop in exited from. cnt jumps to the start of the loop.

loop: while true {
    while true {
        brk loop;
    }
}

For Loop

Do loops in Interp are unlike in other languages. They are simply a syntatic sugar.

Let's define an iterator function:

fun Range(function Function<Int, Void>, from Int, to Int) {
    var i = from - 1;
    while i < to {
        i += 1; # make the loop operation before the function call statement
                # so that the continue statement works as intended
        function(i);
    }
}

These two are identical!

do i in Range(0, 10) {
    Print(i);
}

# output:
# 0
# 1
# 2
# etc.

fun LoopBody(i Int) {
    Print(i);
}
Range(LoopBody, 0, 10);

Types

The following simple types are pre-defined:

  • Any type matches any value except Void.
  • Void type matches Void.
  • Null type matches Null.
  • String type matches Strings.
  • Int type matches Int.
  • Float type matches Floats.
  • Bool type matches Booleans.
  • List type, read more here.
  • Function type, read more here

And and Or Types

And and Or Types are also allowed

1 is Int or String # true

Structs

Structs define a type that matches any object that has the specified fiels and types.

struct Dog {
    breed String,
    name String,
    age Int[0..]
}

pub Dog(breed String, name String, age Int[0..]) Dog {
    ret { breed=breed, name=name, age=age };
}

Enums

An enum defines a type that can match any of the types given.

enum Number {
    Int,
    Float
}

fun Add(x Number, y Number) Number {
    ret x + y;
}

Atoms

Atoms behave as types and values at the same time.

atom Car;
atom Plane;

for Travel(method Car or Plane) {
    ..
}

Modules

Modules group code into namespaces.

mod math {
    
    fun Sin(n Float) Float {
        ...
    }
    
    fun Cos(n Float) Float {
        ...
    }

}

Module members can be accessed anywhere with the full path specified.

Print(math::Sin(2.3));

Use statements (unique to each file) can be used to bring module members into the global namespace.

use math;
Print(Sin(2.3));

Thread

Interp supports real multi-threading with the ability to send messages to and from threads.

Simple Use

var thread = do Thread() {
    var i = 0;
    var sum = 0;
    while i < 1000 {
        sum += i;
    }
    ret sum;
}

var sum = Join(thread);
Print(sum);

Separate Virtual Machines

Each thread in Interp has it's own virtual machine. There is no GIL like in Python or Ruby. This means that memory is NOT shared! You will have to import all libraries separately.


imp "lib.it";

var thread = do Thread() {
    # this is needed here to use functions from 'lib.it' within the thread
    imp "lib.it";
}

Thread Arguments

var thread = do x, y in Thread(1, 2) {
    if x == 1 & y == 2 {
        Print("Yes, they are equal!");
    }
}

Join(thread);

Messags

Threads can send messages with eachother. Parent-thread argument comes before thread arguments.

var thread = do parent_thread in Thread() {
    Send(parent_thread, "Hello from thread!");
    var message_from_parent = Recv(parent_thread);
    Print(message);
}

var message_from_thread = Recv(thread);
Print(message_from_thread);
Send(thread, "Hello from parent!");
Join(thread);

Import

The imp statement can be used to include Interp code.

imp_this.it:

Print("Hello");

main.it:

imp "imp_this.it"
# output: Hello

You can also import a String variable. Note: Imports are cached, you can't import the same file twice.

Comments

Comments are single-line and start with #

Print("Hello!") # comment

For debugging, Interp offers the Breakpoint function. It spawns a REPL anywhere you want.

var i = 1;

Foreign Functions

Pure Rust functions can be called from Interp.

ImportForeign

This function can be used to import a foreign function.

var MyForeignFunc = ImportForeign(
    "path_to_shared_object",
    "ForeignFunc"
);

MyForeignFunc();

Creating a Foreign Function

Start by creating a cargo project.

$ cargo new extend_interp_lib --lib

Edit the Cargo.toml:

[package]
name = "interp_extend_lib"
version = "0.1.0"
edition = "2021"

[dependencies]
interplang = { path = "../../interp/interplang" } # Path to interplang

[lib]
crate-type=["dylib"] # Make a dynamic rust library

Edit src/lib.rs:

#![allow(unused)]
fn main() {
use interplang::vm::value::Value;
use interplang::parser::loc::Loc;

#[no_mangle]
pub fn Interp_Hello(args: Vec<(Value, Loc)>, named_args: Vec<(String, Value, Loc)>, loc: Loc) -> Value {
    Value::String("Hello".to_owned())
}
}

Then you can import it with ImportForeign:

pub HelloFunction = ImportForeign(
    "target/debug/libinterp_extend_lib.so",
    "Hello"
);

Print(HelloFunction());
# outpuit: Hello
